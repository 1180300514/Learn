不同操作系统中，串口的编号方式不同：
在Windows系统中，串口名称为"COM"加数字编号，如COM7；
在Mac OS中串口名称为"/dev/cu.usbmodem"加数字编号；
在Ubuntu中串口名称为"/dev/ttyACM"加数字编号。

pinMode(13,OUTPUT);
digitalWrite(13,HIGH);
pinMode(13,OUTPUT)即是设置引脚的模式，这里设定了13脚为输出模式；而digitalWrite(13,HIGH) 是让13脚输出高电平数字信号。

Arduino程序基本结构由setup() 和loop() 两个函数组成：

void setup()
Arduino控制器通电或复位后，即会开始执行setup() 函数中的程序，该部分只会执行一次。

void loop()
在setup() 函数中的程序执行完后，Arduino会接着执行loop() 函数中的程序。而loop()函数是一个死循环，其中的程序会不断的重复运行。

实际开发中，通常在setup() 函数中完成Arduino的初始化设置，如配置I/O口状态，初始化串口等操作；在loop() 函数中完成程序的主要功能，如驱动各种模块，采集数据等。

在使用输入或输出功能前，需要先通过pinMode() 函数配置引脚的模式为输入模式或输出模式。

pinMode(pin, mode);
参数pin为指定配置的引脚编号；参数mode为指定的配置模式。
INPUT	输入模式
OUTPUT	输出模式
INPUT_PULLUP	输入上拉模式

数字引脚除了用于输出信号外，还可以用digitalRead() 函数读取外部输入的数字信号，其调用形式为：

int value = digitalRead(pin);
参数pin为指定读取状态的引脚编号；返回值value为获取到的信号状态，1为高电平，0为低电平。

存储空间（Memory）：
“标准”Arduino通常有两个存储器：SRAM(静态随机存取存储器)和FLASH(闪存)

SRAM:   用于存储变量的值（例如布尔值的状态）。关机后，此内存将重置。
FLASH:  主要用于存储主程序或微控制器的指令。该存储器在断电时不会被擦除，因此在电路板通电后立即执行微控制器的指令。

Arduino上可用的内存量因板而异。例如，Arduino UNO 具有 32kB 闪存/2kB SRAM，而 Nano 33 IoT 具有 256kB 闪存/32kB SRAM。您可以在每个产品的文档页面中找到此信息，这些页面可在Arduino硬件文档中找到。

其中0.5KB分作BOOT区用于储存引导程序，实现串口下载程序的功能，另外的31.5KB是用户可以储存程序的空间。相对于现在动辄几百G的硬盘，可能觉得32KB很小很小，但在单片机上，32KB已经可以存储很大的程序了。

EEPROM：1KB
EEPROM全称为电可擦写可编程只读存储器，是一种用户可更改的只读存储器，其特点是Arduino断电或者复位后，其中的数据不会丢失。

VS里的PIO在用包时候在ini文件里加一句话就可以不用单独找包下载包，想知道这句话在哪可以去Libraries里面找Installation模块，里面一般有三个版本，以ESP32epd为例：

1、gooddisplayshare/ESP32epd @ ^1.0.1: 这个符号 ^ 表示该库的版本为 1.0.1 或任何向前兼容的版本（即大于或等于 1.0.1，但小于 2.0.0）。如果未来有 1.0.2、1.0.3 等更新版本，这个约束会自动选择这些版本。
2、gooddisplayshare/ESP32epd @ ~1.0.1: 这个符号 ~ 表示该库的版本为 1.0.1 或任何向前兼容的小更新版本（即大于或等于 1.0.1，但小于 1.1.0）。这意味着只有 1.0.1 和 1.0.2 会被接受，但不包括 1.1.0 及更高版本。
3、gooddisplayshare/ESP32epd @ 1.0.1: 这是一个精确版本号，只会安装 1.0.1 这个版本。

如果您希望最大限度地保持与未来的更新兼容，并且希望自动获取小版本更新，可以选择 gooddisplayshare/ESP32epd @ ^1.0.1。
如果您只想使用 1.0.x 系列中的特定版本，并且不想自动更新到未来的次要版本，那么可以选择 gooddisplayshare/ESP32epd @ ~1.0.1。
如果您希望确保您的项目始终使用确切的版本，而不会因未来的更新而引入不兼容的变化，您可以选择 gooddisplayshare/ESP32epd @ 1.0.1。

一般来说，推荐使用 gooddisplayshare/ESP32epd @ ^1.0.1，以便能够接收到向后兼容的小版本更新，从而获得 bug 修复和改进，同时又不必担心大的不兼容变化。如果您有特殊需求，也可以根据您的情况选择其他选项。
